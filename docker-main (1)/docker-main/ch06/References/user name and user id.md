리눅스에서 **user name**(사용자 이름)과 **user ID**(UID)는 서로 다른 개념이지만 밀접하게 연관되어 있습니다. 유닉스 계열 시스템의 사용자 관리 방식에서 비롯된 개념입니다.

---

## 1. User Name (사용자 이름)

* **사람이 읽기 쉬운** 사용자의 논리적 이름입니다.
* 예: `root`, `alice`, `bob`, `student`
* 로그인할 때, 쉘 프롬프트에서, 또는 `ls -l` 같은 명령으로 파일 소유자를 볼 때 **이름**이 보입니다.
* `/etc/passwd` 파일에서 사용자 계정에 대한 정보를 다음과 같이 확인할 수 있습니다. 

  ```
  alice:x:1001:1001:Alice:/home/alice:/bin/bash
  ```

  여기서 `alice`가 바로 **user name**입니다.

---

### 필드별 의미

| 순번 | 필드 값          | 의미                         | 상세 설명                                                              |
| -- | ------------- | -------------------------- | ------------------------------------------------------------------ |
| 1  | `alice`       | **사용자 이름(User Name)**      | 사람이 로그인할 때 쓰는 계정명. 시스템 내부적으로는 UID를 기반으로 동작함.                       |
| 2  | `x`           | **패스워드(Password)**         | 실제 암호는 `/etc/shadow` 파일에 해시 형태로 저장됨. `/etc/passwd`에는 단순히 `x`로 표시됨. |
| 3  | `1001`        | **사용자 ID(UID)**            | 시스템이 사용자를 식별하는 숫자 ID. 0은 root, 1\~999는 시스템 계정, 1000부터 일반 사용자.      |
| 4  | `1001`        | **그룹 ID(GID)**             | 사용자가 기본으로 속한 그룹의 ID. 그룹 정보는 `/etc/group`에서 확인 가능.                  |
| 5  | `Alice`       | **GECOS 필드**               | 사용자의 설명 또는 전체 이름. 종종 비어 있을 수도 있음.                                  |
| 6  | `/home/alice` | **홈 디렉터리(Home Directory)** | 사용자의 기본 작업 디렉터리. 로그인 시 진입하는 기본 위치.                                 |
| 7  | `/bin/bash`   | **로그인 셸(Login Shell)**     | 로그인 시 실행되는 기본 셸 프로그램. 여기서는 `bash` 셸.                               |

---

## 2. User ID (UID)

* **숫자로 된 사용자 식별자**입니다.
* 커널은 사용자를 이름이 아니라 UID로 인식합니다.
* 예: `0`(root), `1000`, `1001`, ...
* UID는 시스템 내부적으로 사용되며, 파일 소유권이나 권한 체크 시 UID를 기반으로 합니다.

**주요 UID 범위**:

* **0** → root (슈퍼유저)
* **1\~999** → 시스템/데몬 계정
* **1000 이상** → 일반 사용자 계정(배포판마다 기본값이 조금씩 다를 수 있음)

---

## 3. 매핑 관계

* user name과 UID는 **1:1 매핑**됩니다.
* user는 name으로 로그인하지만, 시스템은 이를 UID로 변환해 내부적으로 처리합니다.
* `id` 명령을 실행하면 다음처럼 표시됩니다:

  ```
  $ id alice
  uid=1001(alice) gid=1001(alice) groups=1001(alice),27(sudo)
  ```

  여기서 `uid=1001(alice)`가 이름과 UID의 매핑을 보여줍니다.

---

## 4. 실제 예시

* 로그인 시:

  ```
  login: alice   → UID 1001로 변환
  ```
* 파일 접근 시:

  ```
  -rw-r--r-- 1 alice alice 1234 Sep 17  test.txt
  ```

  내부적으로는 UID=1001, GID=1001 기반으로 권한을 체크합니다.

---

## 5. 요약 표

| 구분    | User Name        | User ID (UID) |
| ----- | ---------------- | ------------- |
| 형태    | 사람이 읽는 이름        | 정수 값          |
| 사용 위치 | 로그인, 화면 출력       | 시스템 내부 권한 체크  |
| 예시    | root, alice, bob | 0, 1000, 1001 |
| 저장 위치 | /etc/passwd      | /etc/passwd   |
| 고유성   | 이름 중복 가능(비추천)    | UID는 반드시 유일   |
