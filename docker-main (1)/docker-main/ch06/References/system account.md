리눅스 시스템에는 설치 직후부터 디폴트로 생성되는 여러 **시스템 계정(system account)** 들이 있습니다. 이 계정들은 일반 사용자의 로그인을 위한 것이 아니라, 시스템을 위해 존재합니다.

---

## 1. 주요 기본 계정들

| 계정명            | UID 범위      | 역할 / 특징                                           | 로그인 가능 여부 |
| -------------- | ----------- | ------------------------------------------------- | --------- |
| **root**       | 0           | 최고 권한 관리자 계정. 모든 작업 가능.                           | 가능        |
| **daemon**     | 일반적으로 1     | 백그라운드 데몬 실행용 계정.                                  | 불가        |
| **bin**        | 2           | 오래된 유닉스 시스템에서 바이너리 실행 전용 계정(현대 리눅스에서는 거의 사용 안 함). | 불가        |
| **sys**        | 3           | 시스템 프로세스 계정.                                      | 불가        |
| **sync**       | 4           | `sync` 명령 실행 시 사용되던 계정 (예전 시스템 유산).               | 불가        |
| **games**      | 5           | 게임 실행 전용 계정(레거시).                                 | 불가        |
| **man**        | 6           | 매뉴얼(man pages) 관련 계정.                             | 불가        |
| **lp**         | 7           | 프린터 서비스(라인 프린터, Line Printer)용 계정.                | 불가        |
| **mail**       | 8           | 메일 서비스 데몬 전용 계정.                                  | 불가        |
| **www-data**   | 배포에 따라 다름   | Apache/Nginx 같은 웹 서버 실행 전용 계정.                    | 불가        |
| **nobody**     | 일반적으로 65534 | 최소 권한 계정(가장 낮은 권한, 파일 접근 권한 거의 없음).               | 불가        |
| **operator**   | 배포에 따라 다름   | 일부 BSD 계열 유산 계정, 시스템 운영 보조용.                      | 불가        |
| **systemd-**\* | 배포에 따라 다름   | systemd 서비스 전용 계정들 (예: `systemd-network`).        | 불가        |

**로그인 불가 계정**은 보통 `nologin` 또는 `/bin/false` 쉘을 가지며, 직접 로그인은 차단되고 데몬만 실행하도록 제한됩니다.


---

## 2. nobody 계정의 특별한 역할

* UID: 보통 **65534**
* 권한이 거의 없는 사용자로, 보안적으로 가장 안전한 계정
* 웹 서비스나 테스트용으로 **권한 없는 실행 환경**을 만들 때 자주 사용됨
