# 🕒 리눅스 `date` 커맨드

리눅스의 `date` 명령어는 단순히 현재 시간을 출력하는 도구처럼 보이지만, 실제로는 시스템 관리, 로그 처리, 자동화 스크립트, DevOps 파이프라인 등에서 핵심적인 역할을 합니다.

---

## 1. `date` 명령어의 핵심 개념

`date` 명령어는 단순 출력 외에도 다음과 같은 역할을 합니다:

* 현재 시간 표시 및 포맷팅
* 특정 시간대(Time Zone) 기반 시간 출력
* 날짜 연산(하루 전, 한 시간 후 등)
* 시스템 시간 변경(관리자 권한 필요)

```bash
# 기본 사용
$ date
Thu Sep  4 19:30:02 KST 2025
```

---

## 2. 고급 포맷팅: `+` 옵션과 서식 지정자

`date`의 가장 강력한 기능 중 하나는 **사용자 정의 포맷**입니다.
`+` 뒤에 **포맷 문자열**을 지정하면 원하는 형태로 시간과 날짜를 출력할 수 있습니다.

| 서식   | 의미                | 예시 출력      |
| ---- | ----------------- | ---------- |
| `%Y` | 연도(4자리)           | 2025       |
| `%m` | 월(01-12)          | 09         |
| `%d` | 일(01-31)          | 04         |
| `%H` | 시(00-23)          | 19         |
| `%M` | 분(00-59)          | 30         |
| `%S` | 초(00-59)          | 02         |
| `%Z` | 타임존               | KST        |
| `%s` | Unix Timestamp(초) | 1736057402 |

```bash
# yyyy-mm-dd HH:MM:SS 형식 출력
$ date "+%Y-%m-%d %H:%M:%S"
2025-09-04 19:30:02
```

---

## 3. 타임존(Time Zone) 관리

`date` 명령어는 환경 변수 \*\*`TZ`\*\*를 통해 타임존을 동적으로 바꿀 수 있습니다.
시스템 시간을 바꾸지 않고 특정 타임존 기준 시간을 출력할 때 유용합니다.

```bash
# 뉴욕 시간 출력
$ TZ="America/New_York" date
Thu Sep  4 06:30:02 EDT 2025
```

타임존 데이터는 보통 `/usr/share/zoneinfo`에 저장되며, `tzselect` 명령어로 쉽게 설정할 수 있습니다.

---

## 4. 날짜 연산: `-d` 옵션

`-d`(또는 `--date`) 옵션은 **미래·과거 날짜 계산**에 자주 사용됩니다.

```bash
# 내일 날짜
$ date -d "tomorrow" "+%Y-%m-%d"
2025-09-05

# 7일 전 날짜
$ date -d "7 days ago" "+%Y-%m-%d"
2025-08-28

# 다음 주 월요일
$ date -d "next monday" "+%Y-%m-%d"
2025-09-08
```

---

## 5. Unix Timestamp 다루기

서버 로그, DB 타임스탬프 등에서는 \*\*Epoch Time(1970-01-01부터 경과한 초)\*\*를 많이 사용합니다.

```bash
# 현재 시간 → 타임스탬프
$ date +%s
1736057402

# 타임스탬프 → 사람 읽을 수 있는 시간
$ date -d @1736057402 "+%Y-%m-%d %H:%M:%S"
2025-09-04 19:30:02
```

---

## 6. 시스템 시간 변경 (root 전용)

서버 유지보수 중에는 시스템 시간을 직접 변경할 때가 있습니다.

```bash
# 시스템 시간을 2025-09-04 20:00:00으로 변경
$ sudo date -s "2025-09-04 20:00:00"
```

주의: NTP 동기화 중인 서버에서는 변경 사항이 곧바로 덮어써질 수 있습니다.
보통 `timedatectl`과 같은 현대적인 명령어를 권장합니다.

---

## 7. 스크립트 활용 사례

로그 파일 자동화, 백업 스크립트에서 `date`는 자주 쓰입니다.

```bash
#!/bin/bash
# backup_2025-09-04.tar.gz 형태로 백업 파일 생성
BACKUP_FILE="backup_$(date '+%Y-%m-%d').tar.gz"
tar -czf /backup/$BACKUP_FILE /data
```

또는 CI/CD 파이프라인에서 **배포 버전 태깅**에 활용하기도 합니다.

```bash
VERSION="v$(date '+%Y%m%d%H%M')"
echo "Deploying version: $VERSION"
```

---

## 8. `date` vs `timedatectl`

| 기능        | `date` | `timedatectl` |
| --------- | ------ | ------------- |
| 시간 출력     | ✅ 가능   | ✅ 가능          |
| 시스템 시간 변경 | ✅ 가능   | ✅ 가능          |
| NTP 설정    | ❌ 불가능  | ✅ 가능          |
| GUI 통합    | ❌ 불가능  | ✅ 가능          |

최근 시스템에서는 `timedatectl`을 통해 시간 동기화와 설정을 함께 관리하는 것이 일반적입니다.

---

## 9. 실전 팁: DevOps & 로그 분석

* **로그 필터링**: `grep "$(date '+%Y-%m-%d')"`로 오늘 로그만 추출
* **Cron 스케줄**: `date`를 써서 동적 파라미터 생성
* **CI/CD 태깅**: `date`를 버전명에 포함해 재현성 확보

## 10. 요약
