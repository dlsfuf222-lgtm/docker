“프로비저닝 시스템(provisioning system)”이라는 개념은 클라우드 컴퓨팅, DevOps, 네트워크 관리, 심지어는 IoT 환경까지 다방면에서 쓰이는데요. 단순히 “시스템 설치 자동화” 정도로 끝나는 개념이 아니라, **자원 할당 → 환경 구성 → 소프트웨어 배포 → 정책 적용**까지 전 주기를 관리하는 핵심 개념입니다.

---

# ⚙️ 프로비저닝 시스템(Provisioning System) 

## 1. 개념 정의

\*\*프로비저닝(Provisioning)\*\*이란 IT 인프라 자원과 서비스를 **사용 가능 상태로 준비**시키는 과정을 의미합니다.
즉, 프로비저닝 시스템은 서버, 스토리지, 네트워크, 소프트웨어 등을 **자동으로 생성·구성·배포**하고 필요한 **정책과 접근 제어**를 적용하는 **자동화 플랫폼**을 말합니다.

```text
사용자 요청 → 프로비저닝 시스템 → 인프라/서비스 자동 생성 → 운영 환경 제공
```

---

## 2. 프로비저닝이 필요한 이유

| 필요성             | 설명                                                 |
| --------------- | -------------------------------------------------- |
| **속도**          | 수작업 서버 세팅 대신 자동화로 몇 분 내 배포 가능                      |
| **일관성**         | 모든 환경(개발·테스트·운영)에서 동일한 설정 보장                       |
| **확장성**         | 트래픽 증가 시 자동 스케일 아웃/인                               |
| **DevOps 필수요소** | CI/CD 파이프라인에서 코드 기반 인프라 관리(Infrastructure as Code) |

---

## 3. 프로비저닝 시스템의 동작 흐름

```text
1. 요청(Request)   → API, 포털, CLI 등을 통해 프로비저닝 요청 발생
2. 계획(Planning)  → 필요한 리소스/환경 파악
3. 실행(Execution) → VM, 컨테이너, 네트워크, 스토리지 생성
4. 구성(Configuration) → OS 패키지 설치, 미들웨어 세팅, 보안 정책 적용
5. 검증(Validation) → 서비스 정상 동작 여부 확인
6. 운영(Operation)  → 모니터링, 스케일링, 복구 관리
```

---

## 4. 구성 요소

| 구성 요소        | 설명                                       |
| ------------ | ---------------------------------------- |
| **프로비저닝 엔진** | 자동화 로직 실행, API 기반 리소스 관리                 |
| **구성 관리 도구** | Ansible, Puppet, Chef 등을 통해 설정 자동화       |
| **오케스트레이션**  | Terraform, CloudFormation으로 인프라 코드화(IaC) |
| **카탈로그/포털**  | 사용자가 셀프서비스 형태로 리소스 요청 가능                 |
| **정책/보안 모듈** | 접근 제어, 보안 그룹, 컴플라이언스 정책 적용               |

---

## 5. 대표적인 프로비저닝 시스템/도구

| 도구/서비스                 | 주요 특징                         | 사용 사례                   |
| ---------------------- | ----------------------------- | ----------------------- |
| **Terraform**          | 코드 기반 인프라(IaC), 멀티 클라우드 지원    | AWS, GCP, Azure 인프라 자동화 |
| **Ansible**            | 에이전트리스(agentless), YAML 기반 설정 | 서버 설정 관리, 패키지 설치 자동화    |
| **Puppet/Chef**        | 대규모 서버 환경에 적합                 | 데이터센터, 온프레미스 인프라 관리     |
| **AWS CloudFormation** | AWS 전용 IaC 도구                 | VPC, EC2, RDS 자동 프로비저닝  |
| **Kubernetes**         | 컨테이너 오케스트레이션                  | Pod, Deployment 자동 관리   |

---

## 6. 클라우드에서의 프로비저닝

클라우드 환경에서는 **IaaS, PaaS, SaaS** 모두 프로비저닝 개념을 사용합니다.

* **IaaS**: VM, 네트워크, 스토리지 자동 생성 (예: EC2, VPC)
* **PaaS**: 애플리케이션 런타임 환경 자동화 (예: Heroku, Elastic Beanstalk)
* **SaaS**: 계정 생성, 라이선스 할당 자동화 (예: Google Workspace, Microsoft 365)

---

## 7. 실전 예시: Terraform + Ansible 파이프라인

```bash
# 1. Terraform으로 AWS EC2 인스턴스 생성
terraform apply

# 2. Ansible로 EC2 인스턴스에 Nginx 설치
ansible-playbook -i inventory install_nginx.yml
```

결과적으로, 몇 분 만에 **서버 생성 → 환경 설정 → 웹 서버 배포**가 모두 끝납니다.

---

## 8. 프로비저닝 vs 배포(Deployment)

| 항목    | 프로비저닝(Provisioning)        | 배포(Deployment)             |
| ----- | -------------------------- | -------------------------- |
| 초점    | 인프라/환경 준비                  | 애플리케이션 코드 배포               |
| 도구 예시 | Terraform, Ansible, Puppet | Jenkins, ArgoCD, Spinnaker |
| 실행 시점 | 환경이 필요할 때                  | 코드가 변경될 때                  |

즉, **프로비저닝은 인프라 준비**, **배포는 코드 배포**에 초점을 둡니다.

## 9. 한눈에 보는 요약

| 키워드          | 설명                                 |
| ------------ | ---------------------------------- |

| 대표 도구        | Terraform, Ansible, CloudFormation |
