# 🚢 도커(Docker) 인트로 – 개발과 배포를 바꾸는 기술

## 1. 왜 도커인가? 🤔

소프트웨어 개발은 단순히 코드를 작성하는 것에서 끝나지 않습니다.
개발 환경, 라이브러리 버전, 운영체제 차이 때문에 **“내 PC에서는 잘 되는데, 서버에서는 안 돼요!”** 라는 말이 자주 나오죠.

도커는 이런 문제를 해결하기 위해 등장했습니다.
도커는 애플리케이션과 그 실행 환경을 **컨테이너(container)** 라는 독립된 단위로 묶어 배포할 수 있도록 도와줍니다.

즉, 개발자가 만든 그대로 → 테스트 서버 → 운영 서버까지 **환경 차이 없는 실행**을 보장하는 것이죠.

---

## 2. 도커가 제공하는 핵심 아이디어 💡

도커는 운영체제 위에서 동작하는 **가상화 기술**의 일종이지만, 전통적인 VM(Virtual Machine)과는 차이가 있습니다.

* 🖥️ **VM (가상머신)**: 운영체제를 통째로 가상화 → 무겁고 느림
* 📦 **Docker (컨테이너)**: 커널은 공유하고, 필요한 앱 + 라이브러리만 격리 → 가볍고 빠름

결국, 도커는 **가볍고 효율적인 가상화** 방식으로,
“애플리케이션 실행 환경을 코드처럼 관리한다”는 개념을 가능하게 합니다.

---

## 3. 도커의 주요 개념 🧩

도커를 이해할 때 반드시 알아야 할 세 가지 키워드가 있습니다.

1. **이미지(Image)**

   * 앱을 실행하기 위한 설계도이자 템플릿.
   * 예: `python:3.10`, `mysql:8.0`

2. **컨테이너(Container)**

   * 이미지를 실제로 실행한 인스턴스.
   * 프로세스가 격리된 독립 공간.

3. **레지스트리(Registry)**

   * 이미지를 저장하고 공유하는 저장소.
   * 대표적으로 **Docker Hub**.

---

## 4. 도커를 사용하면 무엇이 좋아질까? 🚀

* **개발 환경 통일**: 로컬, 테스트 서버, 운영 서버 환경이 동일.
* **빠른 배포**: “이미지 빌드 → 배포”만으로 앱을 즉시 실행.
* **확장성**: 마이크로서비스 아키텍처(MSA) 구현에 최적.
* **이식성**: 리눅스, 맥, 윈도우 어디서든 컨테이너 실행 가능.

결국 도커는 **“애플리케이션 = 코드 + 환경”** 이라는 철학을 실현합니다.

---

# ✨ 마무리

도커는 단순히 “개발자 도구”가 아니라, **현대 소프트웨어 개발의 표준 인프라**로 자리 잡았습니다.
이제는 “도커를 몰라도 된다”가 아니라, \*\*“도커 없이는 협업과 배포가 불가능하다”\*\*는 말이 나올 정도죠.

앞으로의 수업에서는 도커의 기초부터 실전 활용까지 단계적으로 배워,
여러분이 만든 애플리케이션을 **누구나, 어디서든, 동일하게 실행**할 수 있도록 만들어봅시다! 🚀
