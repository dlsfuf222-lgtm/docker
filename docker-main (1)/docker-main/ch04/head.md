리눅스 **`head`** 커맨드는 텍스트 파일이나 표준 입력의 **앞부분 일부를 출력**하는 명령어입니다. 로그 파일, 데이터 파일, 설정 파일을 빠르게 확인할 때 매우 유용합니다

---

## 1. 기본 사용법

```bash
head [옵션] [파일명]
```

* 기본적으로 **처음 10줄**을 출력합니다.
* 여러 개의 파일을 지정하면 파일 이름을 구분해가며 출력합니다.

예시:

```bash
head /var/log/syslog
```

→ syslog 파일의 처음 10줄을 출력.

---

## 2. 주요 옵션

| 옵션           | 설명                         | 예시                                    |
| ------------ | -------------------------- | ------------------------------------- |
| `-n [줄 수]`   | 출력할 줄 수 지정 (기본값: 10)       | `head -n 20 /var/log/syslog` → 처음 20줄 |
| `-c [바이트 수]` | 출력할 바이트 수 지정               | `head -c 100 file.txt` → 처음 100바이트    |
| `-q`         | 여러 파일 처리 시 파일명 헤더를 출력하지 않음 | `head -q file1 file2`                 |
| `-v`         | 항상 파일명 헤더를 출력              | `head -v file1 file2`                 |
| `--help`     | 사용법 도움말 출력                 | `head --help`                         |

---

## 3. 표준 입력과 함께 사용

`head`는 파이프(`|`)와 함께 자주 사용됩니다.
예시:

```bash
dmesg | head -n 15
```

→ 커널 메시지 로그의 처음 15줄만 출력.

---

## 4. 실무 활용 예시

### (1) 로그 모니터링

```bash
head -n 50 /var/log/nginx/access.log
```

→ Nginx 접근 로그 처음 50줄 확인.

### (2) 특정 바이트만 보기

```bash
head -c 200 bigfile.txt
```

→ 큰 텍스트 파일의 앞부분 200바이트만 출력.

### (3) 데이터 샘플링

```bash
head data.csv
```

→ CSV 데이터의 처음 10줄로 데이터 구조 파악.

---

## 5. `head` vs `tail`

| 명령어    | 용도        | 기본 동작      |
| ------ | --------- | ---------- |
| `head` | 파일 앞부분 출력 | 처음 10줄 출력  |
| `tail` | 파일 뒷부분 출력 | 마지막 10줄 출력 |

로그 파일을 볼 때 `tail -f`는 실시간 모니터링에 자주 사용되고, `head`는 파일의 시작 부분 구조를 빠르게 파악할 때 유용합니다.
