리눅스의 디렉토리 구조는 \*\*루트(/)\*\*를 최상위로 하는 트리 구조(Tree Hierarchy)로 되어 있습니다. 모든 파일과 디렉토리는 **단일 계층** 안에 배치되며, 각각의 디렉토리는 특정 목적을 가집니다. 이 구조는 \*\*FHS(Filesystem Hierarchy Standard)\*\*라는 표준을 따르며, 주요 디렉토리와 그 역할은 아래와 같습니다.

---

## 1. 루트 디렉토리(`/`)

* 리눅스 파일시스템의 **최상위 루트 디렉토리**.
* 모든 디렉토리·파일은 루트를 기준으로 계층적으로 확장됩니다.
* 다른 파티션이나 장치도 특정 디렉토리에 **마운트**되어 하나의 트리로 보이게 됩니다.

---

## 2. 주요 시스템 디렉토리

| 디렉토리             | 설명                                                                                                    |
| ---------------- | ----------------------------------------------------------------------------------------------------- |
| `/bin`           | **기본 사용자 명령어** 저장 (ls, cp, mv, cat 등). 부팅 후 일반 사용자가 바로 사용할 수 있는 필수 명령어.                               |
| `/sbin`          | **시스템 관리 명령어** (fdisk, fsck, ifconfig 등). 보통 root 권한 필요.                                              |
| `/boot`          | 부팅에 필요한 커널 이미지(vmlinuz), 초기 램 디스크(initrd/initramfs), 부트로더 설정(grub 등) 저장.                              |
| `/dev`           | **디바이스 파일**(하드디스크, 터미널, 프린터 등)을 관리. `/dev/sda`, `/dev/tty`, `/dev/null` 등.                            |
| `/etc`           | **시스템 설정 파일**. `/etc/passwd`, `/etc/fstab`, 네트워크·데몬 설정 등.                                             |
| `/lib`, `/lib64` | 기본 라이브러리(공유 라이브러리, 커널 모듈 등). `/bin`과 `/sbin` 명령이 동작하려면 필수.                                            |
| `/media`         | USB·CD-ROM 같은 **이동식 매체** 자동 마운트 지점.                                                                   |
| `/mnt`           | 관리자가 임시로 **외부 파티션 마운트**할 때 주로 사용.                                                                     |
| `/opt`           | **추가 소프트웨어**(패키지 관리자 외 설치 프로그램) 설치 디렉토리.                                                              |
| `/proc`          | 커널이 제공하는 **가상 파일시스템**. 실행 중인 프로세스와 커널 파라미터 정보 제공.                                                     |
| `/run`           | 런타임 데이터 저장. 부팅 후 프로세스의 PID, 소켓, 임시 파일 등이 위치.                                                          |
| `/sys`           | 커널이 제공하는 **하드웨어·커널 정보 가상 파일시스템**. `/proc`과 유사하지만 sysfs 기반.                                            |
| `/tmp`           | **임시 파일** 저장. 재부팅 시 삭제되는 경우 많음.                                                                       |
| `/usr`           | **사용자 프로그램, 라이브러리, 문서** 등. OS 핵심 외 대부분의 소프트웨어. <br> `/usr/bin`, `/usr/lib`, `/usr/share` 하위 디렉토리로 구성. |
| `/var`           | 로그 파일(`/var/log`), 메일, 프린트 스풀, 캐시 등 **자주 바뀌는 데이터**.                                                   |
| `/home`          | 일반 사용자들의 **홈 디렉토리**(`/home/user1`, `/home/user2` 등). 사용자별 개인 파일 저장.                                   |
| `/root`          | **root 계정 홈 디렉토리**. `/home/root`가 아님에 주의.                                                             |

---

## 3. 가상 파일시스템 디렉토리

* **`/proc`**: 실행 중인 프로세스 정보, CPU·메모리 사용량, 커널 파라미터(`/proc/cpuinfo`, `/proc/meminfo` 등) 제공.
* **`/sys`**: 장치 트리, 커널 모듈 정보(`/sys/class`, `/sys/devices` 등) 제공.
* **`/dev`**: 실제 디바이스 접근 위한 특수 파일 제공.

이들은 **실제 디스크 공간이 아닌 커널이 제공하는 인터페이스**로, 관리·모니터링 시 중요합니다.

---

## 4. 일반적인 계층 구조 예시

```
/
├── bin        → 필수 명령어
├── boot       → 부팅 파일
├── dev        → 장치 파일
├── etc        → 설정 파일
├── home       → 사용자 디렉토리
├── lib        → 라이브러리
├── media      → 이동식 장치
├── mnt        → 임시 마운트
├── opt        → 추가 소프트웨어
├── proc       → 가상 파일시스템(프로세스/커널 정보)
├── root       → root 계정 홈 디렉토리
├── run        → 런타임 데이터
├── sbin       → 시스템 관리 명령어
├── sys        → 가상 파일시스템(장치/커널 정보)
├── tmp        → 임시 파일
├── usr        → 사용자 프로그램, 라이브러리
└── var        → 로그, 캐시, 가변 데이터
```

---

## 5. 핵심 요약

* `/`는 **루트**, 모든 경로의 시작점.
* `/bin`, `/sbin`, `/lib`는 **부팅과 필수 기능**에 필요.
* `/usr`, `/var`, `/home`은 **일반 프로그램·데이터·로그** 저장소.
* `/proc`, `/sys`, `/dev`는 **가상 파일시스템**으로 커널·디바이스 정보 제공.
