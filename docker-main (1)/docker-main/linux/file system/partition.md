리눅스에서 \*\*파티션(partition)\*\*이란 **하나의 물리적 저장장치(예: HDD, SSD, NVMe)를 논리적으로 나누어 독립적인 저장 공간처럼 사용하는 구획**을 의미합니다. 즉, 하나의 디스크를 여러 개의 작은 디스크처럼 쪼개어 각기 다른 용도(예: 운영체제, 데이터 저장, 스왑 등)로 사용할 수 있게 해줍니다.


---

## 1. 파티션의 필요성

* **운영체제 분리**: 예를 들어 `/boot`, `/home`, `/var`를 각각 별도 파티션에 두어 OS 재설치 시 사용자 데이터 보호.
* **멀티 OS 부트**: 리눅스·윈도우를 같은 디스크에 공존시키려면 서로 다른 파티션 필요.
* **성능·안정성**: 로그(`/var`), 스왑(swap), 사용자 데이터(`/home`) 등을 분리하여 장애 영향 최소화.
* **보안**: 읽기 전용 파티션, 크기 제한, 특정 FS 기능 제한 등 보안 설정 가능.

---

## 2. 파티션 vs 파일시스템 vs 마운트

이 세 가지를 자주 혼동하므로 **계층적 관계**를 먼저 이해하는 것이 중요합니다.

```
[물리적 디스크: /dev/sda]
   │
   ├─ [파티션: /dev/sda1, /dev/sda2, …]
   │     └─ [파일시스템: ext4, xfs, btrfs …]
   │           └─ [마운트: /, /home, /data …]
```

* **파티션**: 디스크의 물리적 구획
* **파일시스템**: 해당 파티션 위에 생성된 논리적 데이터 구조
* **마운트(Mount)**: 파일시스템을 리눅스 디렉터리 트리에 연결

예: `/dev/sda1` → `ext4` → `/home` 마운트 → `/home/user1/docs/file.txt` 접근

---

## 3. 파티션 테이블: MBR vs GPT

디스크의 \*\*첫 번째 섹터(섹터 0)\*\*에는 **파티션 테이블**이 있습니다.

| 구분        | MBR(Master Boot Record)        | GPT(GUID Partition Table) |
| --------- | ------------------------------ | ------------------------- |
| 파티션 개수    | 최대 4개 기본 파티션(확장 파티션 사용 시 더 가능) | 사실상 무제한(보통 128개)          |
| 디스크 크기 한계 | 2TB까지 지원                       | 9.4ZB(이론상)                |
| 부트 정보     | 부트로더 1차 코드 포함                  | 별도 EFI 파티션 사용(UEFI)       |
| 복구/백업     | 백업 없음                          | 파티션 테이블 백업 자동 지원          |
| 활용 예      | 구형 BIOS 시스템                    | 최신 UEFI, 대용량 디스크 필수       |

---

## 4. 리눅스 파티션의 주요 종류

리눅스 설치 시 파티션을 목적에 따라 나눌 수 있습니다.

| 파티션            | 설명                                  | 예시 마운트 지점         |
| -------------- | ----------------------------------- | ----------------- |
| **/ (루트)**     | 운영체제 핵심 파일들                         | `/`               |
| **/boot**      | 커널·부트로더 파일(별도 분리 시 부팅 안정성↑)         | `/boot`           |
| **/home**      | 사용자 데이터                             | `/home`           |
| **/var**       | 로그·메일·캐시 등 자주 변하는 데이터               | `/var`            |
| **스왑(Swap)**   | 가상 메모리(물리 RAM 부족 시 디스크 사용)          | 스왑 전용 파티션         |
| **EFI System** | UEFI 부트 로더 저장(GPT 필수)               | `/boot/efi`       |
| **데이터 전용**     | DB, Docker, 대용량 스토리지 등 용도에 맞춰 별도 생성 | `/data`, `/srv` 등 |

---

## 5. 리눅스에서 파티션 장치명 규칙

* **SATA/SCSI/NVMe 디스크 예시**

  * `/dev/sda`, `/dev/sdb` → 첫 번째, 두 번째 디스크
  * `/dev/sda1`, `/dev/sda2` → 첫 디스크의 1번, 2번 파티션
* **NVMe 디스크**

  * `/dev/nvme0n1p1` → NVMe 첫 디스크, 첫 파티션

## 6. 파티션 관리 명령어

### (1) 파티션 생성·변경

```bash
sudo fdisk /dev/sda     # MBR 파티션 관리
sudo gdisk /dev/sda     # GPT 파티션 관리
sudo parted /dev/sda    # 대화형·스크립트 모드 가능
```

### (2) 파티션 정보 확인

```bash
lsblk         # 파티션·마운트·파일시스템 한눈에 보기
fdisk -l      # MBR 기반 정보
parted -l     # GPT 포함 전체 정보
blkid         # UUID, FS 타입
```

### (3) 파티션 포맷 및 마운트

```bash
sudo mkfs.ext4 /dev/sda1       # 파일시스템 생성
sudo mount /dev/sda1 /mnt      # 마운트
sudo umount /mnt               # 언마운트
```

## 7. 고급 개념: LVM, RAID, 암호화

* **LVM(Logical Volume Manager)**

  * 파티션 위에 **논리 볼륨 계층**을 추가해 크기 조정, 스냅샷 등 유연한 관리 가능.
* **RAID**

  * 여러 디스크를 하나처럼 묶어 **성능 향상(RAID 0)** 또는 **안정성 확보(RAID 1, 5, 6)**.
* **암호화(LUKS/dm-crypt)**

  * 파티션 단위로 전체 디스크 암호화 가능.

## 8. 파티션 구조 다이어그램

```
[ /dev/sda (512GB SSD) ]
 ├─ /dev/sda1 (512MB)   : EFI System (/boot/efi)
 ├─ /dev/sda2 (1GB)     : /boot
 ├─ /dev/sda3 (50GB)    : /
 ├─ /dev/sda4 (100GB)   : /home
 ├─ /dev/sda5 (8GB)     : swap
 └─ /dev/sda6 (나머지)  : /data
```

---

## 9. 요약

* 파티션은 **디스크를 논리적으로 나눈 독립 공간**
* **MBR vs GPT**: GPT가 대용량·최신 표준
* 리눅스에서는 `/dev/sda1`, `/dev/nvme0n1p1` 같은 장치명 사용
* `/`, `/home`, `/var`, `/boot`, `swap` 등 **역할별 분리** 권장
* `fdisk`, `parted`, `lsblk`, `blkid` 명령어로 생성·확인·관리
