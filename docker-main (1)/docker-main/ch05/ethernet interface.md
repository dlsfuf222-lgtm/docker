리눅스에서 **이더넷 인터페이스(Ethernet Interface)** 는 컴퓨터를 유선 네트워크에 연결하는 데 사용되는 논리적인 개념입니다. 이는 하드웨어와 소프트웨어를 연결하는 통로라고 생각하시면 이해하기 쉽습니다.

---

### **1. 물리적 이더넷 인터페이스**

이것은 우리가 흔히 아는 **랜카드(NIC)** 와 연결된 것입니다. 컴퓨터 본체에 꽂혀 있는 RJ45 포트를 생각하면 됩니다. 리눅스는 이 하드웨어 장치를 인식하고, **eth0, eth1**과 같은 이름을 붙여 관리합니다.

최근의 리눅스 배포판은 예측 가능한 네트워크 인터페이스 이름(Predictable Network Interface Names)을 사용합니다. 그래서 `eth0` 대신 **enp0s3**처럼 하드웨어 위치를 기반으로 한 이름을 자주 볼 수 있습니다.

이 물리적 인터페이스는 **고유한 MAC 주소**를 가집니다. MAC 주소는 제조 과정에서 하드웨어에 부여되는 전 세계적으로 유일한 식별자이며, 네트워크에서 데이터를 주고받을 때 사용됩니다.

### **2. 가상 이더넷 인터페이스**

이것은 물리적인 하드웨어 없이 소프트웨어적으로 생성되는 인터페이스입니다. 컨테이너, 가상 머신(VM), 또는 특정 네트워크 기능을 구현할 때 사용됩니다.

* **루프백 인터페이스 (`lo`)**: 가장 대표적인 가상 인터페이스입니다. `127.0.0.1`이라는 IP 주소를 가지며, **자기 자신과의 통신**에만 사용됩니다. 외부와 연결되지 않고, 오직 같은 시스템 내의 프로세스들이 서로 통신하거나 네트워크 설정을 테스트할 때 쓰입니다.
* **가상 이더넷 페어 (`veth`)**: 도커 컨테이너에서 가장 중요한 가상 인터페이스입니다. `veth`는 항상 **두 개의 끝(endpoint)을 가진 파이프**와 같습니다. 한쪽 끝은 컨테이너에 `eth0`와 같은 이름으로 할당되고, 다른 한쪽 끝은 호스트 시스템의 브리지 네트워크에 연결됩니다. 이 구조 덕분에 컨테이너가 호스트를 통해 외부 네트워크와 통신할 수 있게 됩니다.
* **브리지 인터페이스 (`br` 또는 `docker0`)**: 여러 개의 이더넷 인터페이스(물리적 또는 가상)를 하나의 논리적 네트워크로 묶어주는 가상 스위치 역할을 합니다. 도커는 컨테이너를 시작할 때 `docker0`라는 기본 브리지를 생성하고, 이 브리지를 통해 모든 컨테이너들이 서로 통신할 수 있도록 연결해 줍니다.

### **정리하자면**

리눅스에서 이더넷 인터페이스는 단순히 하드웨어(랜카드)를 의미하는 것을 넘어, **네트워크 통신을 위한 모든 논리적인 접점**을 의미합니다. 물리적인 랜카드뿐만 아니라, 컨테이너나 가상화 기술을 위해 생성되는 가상 인터페이스들도 모두 이더넷 인터페이스의 범주에 속합니다.

우리가 `ifconfig`나 `ip addr`와 같은 명령어로 확인할 수 있는 `eth0`, `lo`, `docker0` 등은 모두 리눅스 시스템의 네트워크 환경을 구성하는 중요한 인터페이스들입니다.


---
```
ubuntu@ip-172-31-46-172:~$ ip link
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN mode DEFAULT group default qlen 1000
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
2: ens5: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 9001 qdisc mq state UP mode DEFAULT group default qlen 1000
    link/ether 0a:e4:22:f2:3c:21 brd ff:ff:ff:ff:ff:ff
    altname enp0s5
```
```
1: lo: <LOOPBACK,UP,LOWER_UP> ...
2: ens5: <BROADCAST,MULTICAST,UP,LOWER_UP> ...
```

각 부분의 의미를 정리해드리겠습니다.

---

## 1. 인터페이스 이름과 번호

* **1: lo**

  * `1` → 인터페이스 인덱스 번호 (커널 내부에서 각 네트워크 인터페이스를 식별하기 위한 고유 ID)
  * `lo` → 루프백(Loopback) 인터페이스. 자기 자신과의 통신(127.0.0.1) 용도로 항상 존재.

* **2: ens5**

  * `2` → 인덱스 번호. 부팅 시 커널이 순서대로 할당.
  * `ens5` → 예전의 `eth0` 대신 **Predictable Network Interface Names** 규칙에 따라 생성된 이름.

    * `en` → Ethernet(유선랜)
    * `s5` → Slot 5(가상 장치에서는 가상 버스/슬롯 번호 기반)

즉, 예전에는 `eth0`, `eth1`처럼 단순하게 이름이 붙었는데, 지금은 **하드웨어 위치 기반**으로 예측 가능한 이름을 사용합니다.

---

## 2. altname

```
altname enp0s5
```

* **대체 이름(Alternative Name)**
* 예전 규칙에 따라 붙을 수도 있었던 이름을 알려줍니다.
* `enp0s5`는 **PCI 버스 0, 슬롯 5**를 의미합니다.

---

## 3. 상태 정보

* **\<BROADCAST,MULTICAST,UP,LOWER\_UP>**

  * `BROADCAST` → 브로드캐스트 지원
  * `MULTICAST` → 멀티캐스트 지원
  * `UP` → 인터페이스가 활성화됨
  * `LOWER_UP` → 링크가 물리적으로 연결됨

---

## 4. MAC 주소

```
link/ether 0a:e4:22:f2:3c:21
```

* 네트워크 인터페이스의 **하드웨어 주소(MAC)**.
