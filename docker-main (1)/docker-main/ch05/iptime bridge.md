ipTime 공유기 내부에는 여러 개의 **네트워크 인터페이스(포트 + 무선 LAN)** 가 있고, 이걸 **브리지라는 가상 스위치**가 묶어서 하나의 네트워크처럼 동작하게 만드는 겁니다.

---

## 1. ipTIME 공유기 내부 구조 이해

공유기 안에는 보통 이런 구성요소가 있습니다:

* **WAN 포트**: 외부 인터넷과 연결 (보통 NAT 적용)
* **LAN 포트들**: 내부 PC, NAS 등과 연결
* **무선 인터페이스(Wi-Fi)**: 스마트폰, 노트북 등 무선 장치 연결

이 각각이 독립적인 네트워크 인터페이스인데, 브리지를 통해 하나의 **스위치처럼** 묶여야 서로 통신이 가능합니다.

---

## 2. 브리지(Bridge)의 역할

브리지는 **L2(데이터링크 계층)** 장치로, IP 라우팅이 아니라 **MAC 주소 기반**으로 패킷을 포워딩합니다.

ipTIME 공유기에서 브리지는 주로 다음 역할을 합니다:

* **LAN 포트 + 무선 LAN을 같은 브리지 그룹에 묶기**
  → 유선과 무선을 하나의 네트워크처럼 연결
* **브리지 그룹 분리**
  → 예: 게스트 Wi-Fi는 메인 네트워크와 분리해서 서로 통신 못하게 설정

즉, **브리지는 내부 포트들을 묶어서 같은 네트워크로 만들거나 분리하는 스위치 역할**을 수행합니다.

---

## 3. ipTIME에서 브리지 설정 예시

예를 들어:

* **br0**: LAN1\~4 포트 + 무선 Wi-Fi를 묶어서 하나의 네트워크 대역(192.168.0.x)
* **br1**: 게스트 Wi-Fi만 따로 묶어서 인터넷은 되지만 내부 장치와는 단절

이렇게 하면 게스트 Wi-Fi는 내부 NAS나 PC에 접근 못 하고 인터넷만 사용 가능하게 설정할 수 있습니다.

---

## 4. 요약

| 구성 요소          | 브리지 역할                   |
| -------------- | ------------------------ |
| LAN 포트         | 브리지에 묶으면 같은 네트워크 대역 사용   |
| 무선 LAN (Wi-Fi) | 브리지에 포함 시 유선과 같은 네트워크 대역 |
| 게스트 네트워크       | 별도 브리지로 분리 → 내부 장치 접근 차단 |
